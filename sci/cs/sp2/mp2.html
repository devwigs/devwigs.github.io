For all questions, your programs must:

  • use appropriate comments and docstrings;
  • use descriptive variable names;
  • use “Input-Processing-Output” (IPO) program structure;
  • be user-friendly. (Running the program should be easy, even for someone not already familiar with what the program does.)

Create a Python program in your CS10 repl using the file sp2_guess.py. Write an interactive number guessing game that does the following:

  • The computer selects a random integer between 1 and 100.
  • Ask the user to guess the number.
  • The program should then tell the user if their guess is too high or too low.
  • Continue asking for a guess until the user guesses correctly.
  • Ask the user whether to quit or play again.
  • Bonus: Have the program count how many guesses it takes to get the correct number.
  • Below is some sample output with the user’s input shown in blue.

Guess? 50
Too high!

Guess? 20
Too low!

Guess? 40
Too low!

Guess? 47
That's right! You took 4 tries to guess the number.

Play again? [y/n] n
