For all questions, your programs must:

  • use appropriate comments and docstrings;
  • use descriptive variable names;
  • use “Input-Processing-Output” (IPO) program structure;
  • be user-friendly. (Running the program should be easy, even for someone not already familiar with what the program does.)

Create a Python program in your CS10 repl with the file name sp2_letterSquare.py. Write a program that does the following:

  • Ask the user to type a letter. If the user enters something that is not exactly one character long or is not a letter between 'A' and 'Z', the program must keep asking until it receives valid input.
  • The program should then draw a “letter square” in the console as described below.
Enter a letter: 7
That is not a letter!

Enter a letter: E

AAAAAAAAA
ABBBBBBBA
ABCCCCCBA
ABCDDDCBA
ABCDEDCBA
ABCDDDCBA
ABCCCCCBA
ABBBBBBBA
AAAAAAAAA
  • The user’s letter should appear in the middle of the square.
  • The letter preceding the user’s letter should be printed in a 3x3 square surrounding the first letter.
  • The next letter before should be printed in a 5x5 square surrounding the 3x3 square.
  • Continue printing all letters until the largest square consists of the letter 'A'.
  • Note that your program will not be able to print the letters in the order described!
  • It may be helpful to calculate how far each position in the square is from the edge: spots on one of the four edges will always be "A" and spots that are three places from the closest edge will always be "D".
