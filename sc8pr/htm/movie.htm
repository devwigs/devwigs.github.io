sc8pr.misc.media.FFReader Class

Experimental! Added in v2.2.

The Movie class is used to play media files as they are decoded. Movie implements the ondraw event handler to replace the image pixel data each time a new frame is read from the movie file.

This class uses the imageio and imageio-ffmpeg packages as well as their dependencies (numpy, Pillow). Note that sc8pr does not import any packages other than pygame; your program must import the three modules needed for Movie to work as shown in blue in the example below.

from sc8pr import Sketch
from sc8pr.misc.media import Movie
import imageio, numpy, PIL.Image

def onmousedown(mov, ev):
    mov.paused = not mov.paused

def setup(sk):
    mov = Movie("movie.mp4", size=sk.size)
    sk += mov.bind(onmousedown).config(pos=sk.center)

Sketch().play()

Constructor

Movie(src, interval=None, **kwargs): .

src: The path to an FFmpeg-compatible media file to be decoded.

interval: The number of sketch frames between movie frames. If omitted, sc8pr will attempt to calculate it from the movie metadata assuming a sketch frame rate of 60 fps.

kwargs: Additional options to be passed to the FFReader.

Read-Only Property

reader: An FFReader instance created to read the media file. When the movie is played to the end of the file, the reader closes and the reader attribute will be set to None. If the movie is removed from the sketch before playback is finished, you must call the reader’s close method. The image of the final frame will remain.

Assignable Properties

interval: The number of sketch frames to play before updating the image by reading the next movie frame.

paused: A bool specifying whether playback is paused; default False.

Instance Method

restart() → self: Restarts the movie by closing the reader and creating a new FFReader instance using the same arguments as the original one; sets the paused attribute back to False.
