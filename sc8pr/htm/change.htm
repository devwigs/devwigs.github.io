Changes from sc8pr 2

Event Handlers

In sc8pr 3, all event handlers require two arguments, a Graphic instance and a pygame.event.EventType instance. This was true of most handlers in sc8pr 2, but there were a few exceptions (ondraw, onwrap, onbounce, oncollide, onreset).

Except for some of the robotics simulations, Graphic subclasses no longer implement the ondraw event handler. A new method, Graphic.update, has been added to fulfill each graphic object’s internal requirements. The ondraw handler, if any, is called by update. This means that your ondraw event handlers do not need to call a default handler as they did in sc8pr 2. However, if you override the update method, ondraw will no longer be called automatically.

The onreset event used by CostumeImage and its subclasses is no longer used. CostumeImage.update now includes costume status information as part of the event information when it calls ondraw, making onreset unnecessary.

Removed or Replaced Features

These sc8pr 2 modules have been removed or replaced in sc8pr 3 as they have been superceded by newer features.

  • examples: The robotics simulations are now part of the robot package and other examples may be found in the Gallery rather than the sc8pr package itself.
  • misc.effect: This module has been rewritten and is now the effect package.
  • misc.media: This module has been rewritten with a new API and is now the sc8prx package. This new package contains all features that have dependencies other than pygame.
  • misc.plot: Superceded by the plot package.
  • misc.webcache: Rendering of equations can be done with matplotlib through the sc8prx.mpl module.
  • misc.s8v: This module is obsolete due to the rewritten misc.video module. If you need the data from an S8V file to be readable by sc8pr 3, run this program using sc8pr 2.2 to convert it.

This sc8pr 2 class has been deleted from sc8pr 3 as it has been superceded by newer features.

  • PixelData: The functionality of this class is now part of the Image class.

Other Changes

Here is a non-exhaustive list of additional changes that may require sc8pr 2 programs to be modified to work under sc8pr 3.

  • misc.video has been rewitten with a new API.
  • CostumeImage.costumeSequence is now a property instead of a method.
  • geom.transform2d_Gen has been replaced by geom.transform_gen which uses different parameter names and order.
  • geom.transform2d has had the names and order of the parameters changed to be consistent with geom.transform_gen.
  • Image.fromBytes has been changed to Image.frombytes.
  • Graphic.relXY has been renamed Graphic.relPos.
  • The sketch no longer has a capture attribute to record the sketch automatically. You can record by adding the line w.write(sk) to your sketch’s ondraw handler where w refers to a Writer or Video instance.
