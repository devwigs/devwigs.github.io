sc8pr.misc.media.Grabber Class

Added in v2.1 in the sc8pr.misc.video module and moved to sc8pr.misc.media in v2.2. The Grabber class in v2.0 was experimental and does not conform to this API!

This class performs screen captures on Windows or MacOS using the Pillow package. The example below captures a portion of the screen starting at coordinates (100, 100), converts it to a sc8pr.Image instance, and adds the image to the sketch...

# Code in blue is required in sc8pr 2.2 and should be omitted in v2.1.

from sc8pr.misc.media import Grabber
from sc8pr import Sketch, helper_modules
import PIL.ImageGrab

helper_modules(PIL.ImageGrab)

def setup(sk):
    w, h = sk.size
    sk += Grabber([100, 100, w, h]).img.config(pos=sk.center)

Sketch().play()

As of v2.2, sc8pr no longer imports any non-essential modules such as PIL.ImageGrab directly. To use the Grabber class, your program must import PIL.ImageGrab itself and then register the module as a “helper” by calling helper_modules as in the example above.

Constructor

Grabber(rect=None)

rect: The region of the screen to capture. Can be None to capture the entire screen, a 2-tuple or 2-list specifying the size of the rectangle to capture from the upper left, or a 4-list or 4-tuple giving both the upper left coordinates and the size.

Read-only Property

img: The current contents of the specified rectangle as an Image instance. This is the slowest format for screen captures since it requires the most processing. If speed is an issue, use the pil format and convert later using the PixelData class.

pil: The current contents of the specified rectangle as a PIL.Image.Image instance. This is the quickest format for screen captures.

pix: The current contents of the specified rectangle as a PixelData instance. Images captured in Pillow format can be converted later by calling the PixelData constructor.
