sc8pr.sprite.Sprite Class

sc8pr.sprite.CostumeImage Class

The CostumeImage class represents a sequence of images or costumes. The Sprite class, which represents animated bitmap images, combines CostumeImage with the BaseSprite class that controls the sprite’s motion.

In addition to the attributes they inherit from Graphic or BaseSprite, the following additional attributes are defined...

Constructors

Sprite(image, cols=1, rows=1, flip=0, padding=0)

CostumeImage(image, cols=1, rows=1, flip=0, padding=0)

image: An Image or pygame.Surface instance; a str describing the path to an image file; or a list or tuple of such objects to be used at sprite costumes.

cols, rows, flip, padding: For a single image, these arguments are passed to Image.tiles to create costumes from a spritesheet. These arguments are ignored if a sequence of images is provided.

Read-Only Properties

costumeList: A list of Image instances containing the original sequence of costumes.

image: The current costume as a pygame.Surface that has been scaled and rotated to the sprite’s current size and angle.

Assignable Properties

costumeNumber: The index of the sprite’s current costume.

costumeSequence: A tuple specifying the animation sequence in terms of the costume index in the original costumeList.

costumeTime: The number of frames to allow between costume changes; a negative value will play the costumes in reverse order.

cycle: A bool specifying whether to reset the costume number to the begining of the costume sequence when the last costume becomes due to change; default True.

Other Instance Methods

__getitem__(i) → Image: Returns the costume in the current costume sequence with index i.

__len__() → int: Enables the len function to count the number of costumes in the current costume sequence.

costume() → Image: Returns the sprite’s scaled and rotated current costume.

updateCostume(ev) → None: Called by the update method to switch costumes based on the costumeTime. Adds a status attribute to the event object that has one of the following values: 0 (no costume change), 1 (costume has been changed to the next one in the sequence), 2 (costume has been reset at the end of the sequence), -1 (the end of the sequence has been reached and costumeTime has been set to zero).
