sc8pr.sprite.Sprite Class

The Sprite class represents animated bitmap images. Sprite inherits an onwrap event handler from BaseSprite and also overrides ondraw. If overriding these handlers, you may need to call the default handler to ensure the sprites behave correctly.

In addition to the attributes it inherits from Graphic and BaseSprite, the Sprite class defines the following...

Constructor

Sprite(image, cols=1, rows=1, flip=0, padding=0)

image: An Image or pygame.Surface instance; a str describing the path to an image file; or a list or tuple of such objects to be used at sprite costumes.

cols, rows, flip, padding: For a single image, these arguments are passed to Image.tiles to create costumes from a spritesheet. These arguments are ignored if a sequence of images is provided.

Assignable Properties

costumeNumber: The index of the sprite’s current costume.

costumeTime: The number of frames to allow between costume changes.

Read-Only Properties

costumeList: A list of Image instances containing the original sequence of costumes.

image: The current costume as a pygame.Surface that has been scaled and rotated to the sprite’s current size and angle.

Other Instance Methods

__getitem__(i) → Image: Returns the costume in the current costume sequence with index i.

__len__() → int: Enables the len function to count the number of costumes in the current costume sequence.

costume() → Image: Returns the sprite’s current costume.

costumeSequence(seq, n=None) → self: Specifies the order in which to change the costumes when animating.

seq: A sequence of indices referring to the original costume sequence.

n: An index specifying the costumeNumber. If omitted, the current value is retained unless it would result in an IndexError in which case it is reset to zero.
