The BaseSprite class is an abstract class that defines behavious common to sprites. You should not create BaseSprite instances directly but should use one of its subclasses (such as Sprite) instead.

The BaseSprite class defines default event handlers for ondraw and onwrap. If overriding these handlers, you may need to call the default handler to ensure the sprites behave correctly.

In addition to the attributes it inherits from Graphic, BaseSprite defines the following...

Assignable Properties

acc: The acceleration of the sprite in pixels per frame-squared; defaults to None to disable acceleration.

bounce: An int code specifying whether to bounce the sprite from the edge of the sketch. Constants for these codes are defined in the sc8pr module. Defaults to 0 which specifies no action.

drag: A coefficient specifying how much drag force to apply to the sprite motion; default 0.

spin: The number of degrees by which to change the angle property each frame; default 0.

pen: A 2-tuple describing the pen color and thickness to trace the sprites position; default None.

vel: The velocity of the sprite in pixels per frame; default (0, 0).

wrap: An int code specifying what action to take when the sprite leaves the sketch. Constants for these codes are defined in the sc8pr module. Can be set to 0 to specify no action. Defaults to REMOVE.

Instance Methods

circleBounce(cv) → None: Performs bounce operations when necessary; called automatically be the default ondraw event handler.

cv: The sprite’s canvas.

kinematics() → None: Updates the motion of the sprite; called automatically be the default ondraw event handler.

simpleWrap(cv) → None: Performs wrap operations when necessary; called automatically be the default ondraw event handler.

cv: The sprite’s canvas.
