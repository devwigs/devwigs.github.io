expand_lesswhile Loops

Often when we are writing a program to solve a problem, we need to run the same instructions repeatedly. Here is a program that keeps printing random numbers until it gets a number greater than 99.

# Program 's12_ex1.py'

from random import uniform

num = uniform(0, 100)       # Get the first number
while num <= 99:            # LOOP...
    print(num)              #   Print the number
    num = uniform(0, 100)   #   Get a new number
print(num)                  # Print the last number

The while statement (shown above in blue) tells Python to repeat all of the indented instructions as long as a certain condition remains True. The while statement and the indented block that immediately follows it is called a loop.

In this example, the loop is controlled by a Boolean expression, num <= 99. This “condition” will remain True as long as the uniform function returns a number that does not exceed 99. The program will keep printing random numbers until the uniform function eventually returns a number that is greater than 99. This makes the loop condition False and the loop will end. The program will print this last number and then exit.

In Python, indentation is used to structure loops just as it is for if-elif-else blocks. Only the two indented statements in the example are repeated. The final statement in the program is unindented to show that it is not part of the loop. It will only run once, after the loop is finished.

Python allows you to have nested indented blocks, for example an if statement inside a while loop:

# Program 's12_ex2.py'

from random import randint

roll = None
while roll != 7:
    die1 = randint(1, 6)
    die2 = randint(1, 6)
    roll = die1 + die2
    if roll == 2:
        print("Snake eyes!")
    elif roll == 12:
        print("Boxcars!")
    else:
        print("You rolled a {}.".format(roll))
