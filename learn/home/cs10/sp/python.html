  1. Use your IDE (e.g. Thonny, Eclipse, or repl.it) to create a new file in your CS10 folder called s03_turtle.py. Copy and paste the following program into the file. Try to predict what the program will do. Run the program to see if you were right. 
    # Save this program in your CS10 folder as 's03_turtle.py'

from turtle import *

setup(600, 480)
speed(1)
pencolor("red")
penup()
backward(200)
left(45)
pendown()
forward(200)
exitonclick()

The program consists of a sequence of individual instructions that the computer will execute one at a time. Some programming languages use punctuation marks (often a semi-colon) to mark the end of each instruction. Python prefers to use as little punctuation as possible, so each Python instruction is typed on a separate line.

Most of the instructions in this program are function calls. A function is a named instruction that performs a specific task. In Python (and many other languages), you call a function by typing its name, followed by parentheses. Inside the parentheses, you type any additional information that the function needs to perform its task. The data that are passed to the function are called arguments...

setup(600, 480)

This instruction tells Python to run a function named setup, which sets the size of the graphics canvas. This function takes two arguments. The first argument (600) tells setup what we want the canvas width to be; the second argument (480) is the height.

When calling a function, the parentheses are required even if the function does not require any arguments. For example, the penup() and pendown() instructions.

This program is an example of “Turtle Graphics.” Imagine a turtle walking around the screen, drawing as it moves. We can think of this program as instructions that we are giving to the turtle. The forward and backward functions tell the turtle to move a certain distance in pixels. The left and right functions tell the turtle to turn by so many degrees. The penup and pendown and pencolor functions control the pen that the turtle is drawing with.

Python keeps the definitions for many of its functions in separate files called modules. Before your program can use a function, you need to tell Python to import (load) the function definition. The first line of the program above tells Python to import all functions and data located in the turtle module. If you try to call a function that you haven’t imported, an error will occur. (Try running the program after deleting the first line.)

from turtle import *

The asterix at the end of the import statement tells Python to import everything that it finds in the turtle module. Importing functions and data that your program doesn’t need wastes computer memory and processing power. If you only need to use a few of the features of a module, you should import only what you need...

from turtle import forward, backward, left, right

There are a few functions that are so important or so common that Python loads them automatically; you do not need to import them. These functions are called built-ins. One built in function that we used in the last lesson is print.
