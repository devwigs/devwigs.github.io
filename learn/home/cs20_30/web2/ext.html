expand_lessExternal Stylesheets

So far, we have been placing all of our CSS style declarations inside a style element in the head of our XHTML documents. This is known as an internal stylesheet. This is okay when we are creating single webpages. However, many websites contain several linked pages that use the same style information. Internal stylesheets are inefficient because the same style information has to be included in the HTML document for each and every webpage. This means that the same style information is downloaded as part of every page, and if you decide to change the appearance of the website, you need to edit the style information in every webpage.

A better alternative is an external stylesheet. An external stylesheet is a separate file containing CSS style information. Your HTML documents can then link to the CSS file. With the style information in a single file, webpages can be downloaded more quickly, since the style information only has to be downloaded once rather than with every page. Also, you only need to edit a single CSS file to change the appearance of all webpages that link to that file.

To create an external stylesheet, simply cut the CSS declarations from your style element, paste them into a separate file, and save the file with a name ending in .css. Note that CSS files are not XML or HTML files, so you must not include the start-tag and end-tag of the style element. Once you have saved your external stylesheet, you can delete the style element from the head of your HTML document and replace it in the document head with a link element:

<link type="text/css" rel="stylesheet" href="myStyles.css" />

The href attribute is an absolute or relative URL identifying the CSS file.

Note that the link element uses a self-closing tag. Do not keep the CSS declarations as a child text node of the link element. Your CSS should appear in the external stylesheet only.
