Animations

The main difference between the Sprite and Image classes is that sprites are intended to be animated while images are not. In the example below, we have added a spin argument when configuring the sprite to make it rotate 1 degree clockwise per frame. We have also added a vel (velocity) argument to move the sprite 2 pixels right and 1 pixel down each frame. The acc (acceleration) keyword is used to increase the sprite’s downward velocity by 0.01 each frame.

from sc8pr import Sketch, Image, VERTICAL, HORIZONTAL
from sc8pr.sprite import Sprite

def setup(sk):
    sk.bg = Image("sky.png")
    alien = Sprite("alien.png")
    sk += alien.config(
        pos = sk.center,
        width = sk.width/10,
        spin = 1,
        vel = (2, 1),
        acc = (0, 0.01),
        bounce = VERTICAL,
        wrap = HORIZONTAL
    )

Sketch((512,368)).play()

The bounce and wrap arguments determine what happens when the sprite reaches the edge of the sketch. By specifying bounce = VERTICAL, when the sprite hits the top or bottom edges it will bounce back the way it came. By setting wrap = HORIZONTAL, the sprite will disappear from the left or right and reappear from the opposite edge. If we want the same behaviour from all four edges, we could use BOTH rather than HORIZONTAL or VERTICAL.

The wrap argument can also take values REMOVE (default), REMOVE_X, or REMOVE_Y, to indicate that the sprite should be removed from the animation if it completely leaves the sketch area. You can also specify the value wrap = 0 to indicate that no action is to be taken when the sprite leaves the sketch area.
