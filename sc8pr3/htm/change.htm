Changes from sc8pr 2

Dependencies

While sc8pr's core features depend only on pygame, there are additional dependencies for these “extras”:

  • misc.media and misc.movie modules and misc.video.VidZip.decode and encode methods: Encoding and decoding FFmpeg-compatible files.
  • plot.mpl module: Rendering equations with matplotlib.
  • PixelData.pil property: Converting image from pygame to Pillow format.

These additional dependencies (numpy, pillow, imageio, imageio-ffmpeg, and matplotlib) are now included in setup.cfg so that they will be installed automatically when you install sc8pr using pip.

Event Handlers

In sc8pr 3, all event handlers require two arguments, a Graphic instance and a pygame.event.EventType instance. This was true of most handlers in sc8pr 2, but there were a few exceptions (ondraw, onwrap, onbounce, oncollide, onreset).

Graphic subclasses no longer implement the ondraw event handler. A new method, update, has been added to fulfill each graphic object’s internal requirements. The ondraw handler, if any, is called by update. This means that your ondraw event handlers do not need to call a default handler as they did in sc8pr 2. However, if you override the update method, ondraw will no longer be called automatically.

Removed Modules

These sc8pr 2 modules have been deleted from sc8pr 3 as they have been superceded by newer features.

  • examples
  • misc.plot
  • misc.webcache
  • misc.s8v

Modified / Replaced Features

Some features have been changed sufficiently that sc8pr 2 programs may need to be modified to work under sc8pr 3.

  • misc.media and misc.video modules has been rewritten and have a new API.
  • effect.FastDissolve was an alternate version of the Dissolve effect in sc8pr 2 that used a faster algorithm when numpy was available. This class has been removed and Dissolve will use the best algorithm that is available.
  • geom.transform2d_Gen has been replaced by geom.transform_gen which uses different parameter names and order.
  • geom.transform2d has had the names and order of the parameters changed to be consistent with geom.transform_gen.
